Sol, Terra & Luna:
what to use when.

A reference for picking the model and reasoning level for dev work. Built on cost-per-task data, official guidance, and field reports.

Compiled 2026-07-14 · Artificial Analysis · OpenAI docs · GitHub Copilot docs · Theo (t3.gg) · Simon Willison · field reports
TL;DR · the whole deck in one slide

Run a barbell, not a ladder.

Two models cover almost everything. Use Luna for small, well-defined work. Use Sol for hard, long, or ambiguous work. Where you have a reasoning dial (Codex CLI / API), default to Sol medium. Raise to Sol high for anything difficult or longer than ~10 minutes.

Luna

The small stuff

Spot edits, lookups, tests for known code, bulk and batch jobs. About 4x the intelligence-per-dollar of Sol.

Sol

The hard stuff

Debugging, refactors, long agentic runs. Intelligence Index 59 vs Claude Fable 5's 60, at about a third of the cost per task.

Terra

Skip by default. Artificial Analysis finds every Terra configuration matched or beaten by some Sol or Luna configuration on the intelligence-vs-cost frontier. Terra's honest niche: one-model simplicity, and plans where Sol isn't available.

Max and Ultra are break-glass settings, not defaults. Plan with your highest-judgment model, implement with Sol. Always give 5.6 explicit stop conditions (slide 11).

Quick start · pick a pattern and go

Three working setups.

You don't need the full matrix to start. Pick the pattern that matches how you work today.

Pattern 1 · simplest

Daily-drive Sol medium

  1. Set Sol medium for everything.
  2. Raise to high when a task is hard or runs longer than ~10 minutes.
  3. Drop back to medium when it's done.
One model, one setting, no decisions. Medium is OpenAI's default level and the most-cited daily driver in field reports. Costs roughly $1.86 per task on Theo's curve.
Pattern 2 · experienced

Plan on Sol high, work on Luna xhigh

  1. Plan and decompose with Sol high (xhigh for architecture).
  2. Hand each bounded task to a Luna xhigh subagent.
  3. Review every diff yourself before merging.
Raschka: "Luna with extra-high effort may be better and cheaper than Sol with medium effort." Theo: Luna belongs "orchestrated by a smarter agent," not driven directly.
Pattern 3 · Copilot only

No dial: Luna small, Sol hard

  1. Luna for quick questions, small edits, lookups.
  2. Sol for refactors, debugging, agent sessions.
  3. Terra only when Sol isn't on your plan.
Copilot hides the effort dial, so the model picker is the whole decision. Details on slide 10.

Economics · token volume

Cheaper per token ≠ fewer tokens.

The cheaper tiers think out loud more. To finish the same eval suite, Luna emits nearly twice the tokens Sol does. If "minimal tokens" means latency and context budget, Sol wins. If it means dollars, Luna wins. Know which one you're optimizing.

The reasoning dial · Codex CLI / API / ChatGPT (not Copilot)

High is the elbow. Max is a luxury.

Six effort levels (none → low → medium → high → xhigh → max) plus Ultra, a parallel multi-agent mode. Theo's published curve from ~$200k of Sol usage: quality climbs steeply to high, then crawls while cost keeps doubling.

Sol: task score vs cost per task, by effort level

Theo (t3.gg) cost/quality points · secondhand via BigGo, 2026-07-13 · directionally matches OpenAI's own "test one level lower" guidance

OpenAI's own rules: default is medium; raise to high/xhigh "only when evals show a meaningful gain"; "do not recommend max globally"; when migrating from 5.5, test one level lower first. 5.6 holds quality with fewer tokens. Practitioners prefer xhigh over Ultra for steering (fewer subagents); Ultra "exhausts weekly quotas in hours."

Red lines · learned the expensive way

Seven ways 5.6 will burn you.

Never give Luna long-context recall. Retrieval collapses to 41.3% past 256K (Sol: 91.5%). Big-repo needle-finding is Sol's job.
Watch the 272K surcharge. Tokens bill at 2x past 272K context (Luna: 200K) in Codex and Copilot alike. Restart sessions; don't grow threads.
Stop conditions in every agentic prompt. "Write the plan, then stop for feedback." Theo: "It's so unlikely to stop that I have to put up the stop signs myself." Cut his burn 4-5x.
Restrain subagents. Add one line to agent.md: "Only use sub-agents if the user explicitly requests them." It stops autonomous fan-out you didn't order.
Review every diff. METR measured Sol's reward-hacking rate as the highest of any public model it has assessed, with an above-average tendency to exceed user intent.
Don't default to max/Ultra. OpenAI's own guidance: "do not recommend max globally." Ultra costs ~3x for ~3 points and drains weekly quotas in hours.
Migrate downward. Coming from 5.5? Test one effort level lower first. 5.6 usually holds quality with fewer tokens.
